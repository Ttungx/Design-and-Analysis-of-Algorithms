# 题目：合并排序（Merge Sort）

# 一、问题背景

合并排序源于实际的大规模数据排序需求。在数据量较大（如十万级、百万级数据）时，冒泡排序、插入排序等简单排序算法的时间复杂度（O(n²)）会导致效率极低，而合并排序基于分治思想，通过"分拆-合并"的策略将时间复杂度优化至O(nlogn)，适用于对排序效率要求较高的场景，如数据库查询结果排序、大数据分析中的数据预处理等。

合并排序是分治算法（Divide and Conquer）的经典应用，由约翰·冯·诺依曼于1945年提出。分治算法的核心思想是：将一个复杂问题分解为若干个规模较小、结构相同的子问题，分别求解子问题后，将子问题的解合并为原问题的解。

# 二、问题描述

给定一个长度为n的无序整数数组，使用合并排序算法对其进行升序排序，输出排序后的有序数组。

**输入：** 无序整数数组A[0..n-1]，其中n为数组长度（n≥0）

**输出：** 与A元素相同、按升序排列的有序数组B[0..n-1]

**约束条件：**

- 排序过程中需保持元素的稳定性（相等元素的相对位置不变）
- 算法时间复杂度不高于O(nlogn)

# 三、设计思路

合并排序的核心是"分治"策略，分为三个步骤：

1. **分解（Divide）：** 将长度为n的数组拆分为两个长度为n/2的子数组
2. **解决（Conquer）：** 递归调用合并排序，分别对两个子数组进行排序，直至子数组长度为1
3. **合并（Combine）：** 创建临时数组，将两个有序子数组合并为一个有序数组

**关键步骤详解：**
- 递归终止条件：数组长度≤1时，直接返回数组
- 分解逻辑：计算中间索引mid = len(arr) // 2，拆分为左右子数组
- 合并逻辑：使用三指针比较左右子数组元素，按序存入临时数组

**手动模拟示例：** 数组[8, 4, 5, 7, 1, 3, 6, 2]

分拆阶段：
- 第1次拆分：[8,4,5,7] 和 [1,3,6,2]
- 第2次拆分：[8,4]、[5,7]、[1,3]、[6,2]
- 第3次拆分：[8]、[4]、[5]、[7]、[1]、[3]、[6]、[2]（不可再分）

合并阶段：
- 第1次合并：[4,8]、[5,7]、[1,3]、[2,6]
- 第2次合并：[4,5,7,8]、[1,2,3,6]
- 第3次合并：[1,2,3,4,5,6,7,8]（最终有序数组）

**时间复杂度：** O(nlogn)（最坏、最好、平均情况均相同）

**空间复杂度：** O(n)（临时数组为主要开销）
