## 相关知识
二分搜索（Binary Search）是一种高效的搜索算法，适用于在**有序数组**或列表中查找特定元素。其核心思想是通过逐步缩小搜索范围，将时间复杂度降至 **O(log n)**。

### 基本原理
1. **前提条件**：数组或列表必须有序（升序或降序）。
2. **步骤**：
   - 初始化两个指针：`left` 指向起始位置，`right` 指向末尾。
   - 计算中间位置 `mid = left + (right - left) // 2`。
   - 比较中间元素与目标值：
     - 若相等，返回 `mid`。
     - 若中间元素小于目标值，调整 `left = mid + 1`。
     - 若中间元素大于目标值，调整 `right = mid - 1`。
   - 重复上述步骤，直到 `left` 超过 `right`，若未找到则返回 -1。

### 时间复杂度
- **最优**：O(1)（目标值在中间）。
- **最坏**：O(log n)（目标值在边缘或不存在）。
- **平均**：O(log n)。

### 空间复杂度
- **迭代实现**：O(1)。
- **递归实现**：O(log n)（栈空间）。

### 优缺点
- **优点**：
  - 高效，适合大规模数据。
  - 易于实现。
- **缺点**：
  - 要求数据有序。
  - 插入或删除操作成本高。

### 应用场景
- 查找有序数组中的元素。
- 解决数学问题，如平方根计算。
- 在数据库索引中快速定位记录。

### 变种
1. **查找第一个或最后一个匹配项**：调整指针移动条件。
2. **查找最接近的值**：在未找到时返回最接近的值。
3. **在无限序列中搜索**：结合指数搜索。

### 注意事项
- 确保数据有序。
- 注意边界条件，如空数组或单个元素数组。
- 避免整数溢出，使用 `mid = left + (right - left) // 2` 而非 `(left + right) // 2`。
