# 题目：二分搜索技术

# 一、问题背景

二分搜索（Binary Search）是一种高效的搜索算法，适用于在**有序数组**或列表中查找特定元素。其核心思想是通过逐步缩小搜索范围，将时间复杂度降至O(log n)。

在实际应用中，二分搜索广泛应用于查找有序数组中的元素、解决数学问题（如平方根计算）、在数据库索引中快速定位记录等场景。相比线性搜索O(n)的时间复杂度，二分搜索在大规模数据查找中具有显著优势。

# 二、问题描述

输入一个从小到大排序的整数数组a和一个整数x，请找出数组中元素x的位置。要求算法的时间复杂度为O(log n)。

**输入：** 第一行为有序整数数组，第二行为待查找的目标整数x

**输出：** 若找到目标值，输出其索引位置；若未找到，输出"没有这个数"

**约束条件：**
- 数组必须有序（升序排列）
- 算法时间复杂度不高于O(log n)

# 三、设计思路

二分搜索的核心思想是通过不断缩小搜索范围来定位目标元素：

1. **初始化指针**：设置`left`指向起始位置，`right`指向末尾位置
2. **计算中间位置**：`mid = left + (right - left) // 2`（避免整数溢出）
3. **比较与调整**：
   - 若 `arr[mid] == x`，返回 `mid`
   - 若 `arr[mid] < x`，说明目标在右半部分，调整 `left = mid + 1`
   - 若 `arr[mid] > x`，说明目标在左半部分，调整 `right = mid - 1`
4. **终止条件**：当 `left > right`时，搜索结束，未找到目标

**手动模拟示例：** 在数组[1, 2, 3, 4, 5]中查找元素3

- 初始：left=0, right=4
- 第1次：mid=2, arr[2]=3 == 目标值3，返回2

**时间复杂度：** O(log n)（最坏、平均情况），O(1)（最优情况）

**空间复杂度：** O(1)（迭代实现），O(log n)（递归实现）
